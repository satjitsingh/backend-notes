# Summary: Key Takeaways

## Interview Quick Reference

| Topic | Key Points |
|-------|------------|
| **OOP** | Encapsulation, Inheritance, Polymorphism, Abstraction |
| **Collections** | ArrayList vs LinkedList, HashMap internals, TreeMap for sorting |
| **Generics** | Type erasure, PECS, wildcards |
| **Exceptions** | Checked vs Unchecked, try-with-resources |
| **Concurrency** | synchronized, volatile, Executors, CompletableFuture |
| **Streams** | Lazy evaluation, collectors, parallel streams |
| **Memory** | Heap/Stack, GC generations, memory leaks |
| **Java 21** | Virtual threads, records, sealed classes, pattern matching |

## Best Practices Summary

1. **Favor composition over inheritance**
2. **Program to interfaces, not implementations**
3. **Use try-with-resources for all AutoCloseable**
4. **Prefer immutable objects**
5. **Use Optional for nullable returns**
6. **Use executors, not raw threads**
7. **Profile before optimizing**
8. **Override equals, hashCode, and toString**
